What is GridSnap?

GridSnap is a capture tool built for QA engineers who need to document content rendered in Canvas elements, IFRAMEs, or other non-standard layouts. It lets you snap multiple screenshots of a defined area, then automatically stitches them into a single, scrollable grid image.

Quick Start

  1. Navigate to the page you want to capture.
  2. Click Start Capture Session above.
  3. Draw a rectangle over the area you want to capture (the "Focus Zone").
  4. Scroll the page to reveal new content, then press S to snap.
  5. Repeat step 4 as many times as needed.
  6. Press Enter to finish. A preview page will open with your assembled image.

Hotkeys

  • S — Take a snap of the current Focus Zone.
  • B — Force a manual column break. The next snap will start a new column in the output grid.
  • Enter — Finish the session and open the preview.
  • Esc — Cancel the session without saving.

Column Breaks

Snaps stack vertically in the output. When the combined height exceeds the Max Column Height setting, GridSnap automatically wraps into a new column to the right.

You can also press B at any time to force a new column. This is useful for separating logical sections, for example capturing Paytable screens in one column and Standard Rules in the next.

Overlap & Stitching

When scrolling between snaps, you don't need to be precise. GridSnap uses pixel-matching (cross-correlation) to detect overlapping regions between consecutive snaps and removes the duplicated area automatically. Just make sure there is some visible overlap between each scroll.

Tip: For best results, scroll so that roughly 10-30% of the previous snap's content is still visible when you take the next snap.

Preview & Export

After finishing, a preview tab opens showing the full assembled canvas and each individual snap. You can download the entire grid image or any individual snap separately.

Settings

Click Advanced Settings on the Capture tab to access these options:

  • Max Column Height — The pixel height limit before auto-wrapping to a new column. Default is 2000px. Increase for longer single-column output; decrease for a wider, shorter grid.
  • Column Gap — Horizontal spacing between columns in the output image. Default is 20px.
  • Export Format — PNG is lossless but large. JPEG and WebP produce smaller files, useful for bulk uploads to Jira or Confluence.
  • Match Sensitivity — Controls how strict the overlap detection is. If stitching produces visible seams or double-content, try lowering to 0.85.
Note: GridSnap cannot run on browser-internal pages (chrome://, about:, etc.). Navigate to a regular website first.
